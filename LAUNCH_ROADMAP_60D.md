# emre-toast 60-Day Launch Roadmap

Date: 2026-04-02
Primary KPI: npm installs
Secondary KPI: GitHub stars + docs conversion
Positioning: AI-first toast for real-time React apps

## Phase 1 (Days 1-10): Trust Foundation

Goals:

- Ensure every public snippet works copy-paste
- Eliminate API naming inconsistencies
- Make npm page conversion-ready

Execution:

1. README pass:
   - quick start
   - 4 canonical examples (stream, progress, undo, grouping)
2. Docs pass:
   - ensure all pages use same API language
   - add FAQ for common setup problems
3. Package metadata pass:
   - keywords, description, homepage, bugs
4. Demo quality pass:
   - mobile and desktop sanity checks

Success criteria:

- A new user can install and render first toast in < 3 minutes.

## Phase 2 (Days 11-25): Product Story + Content

Goals:

- Build narrative that explains why emre-toast exists
- Create launch assets that can be reused across channels

Execution:

1. Publish 3 core articles:
   - `why existing toasts break in real-time UX`
   - `sonner vs emre-toast`
   - `next.js integration`
2. Create one short demo video (30-45s)
3. Prepare 8-10 screenshots/GIFs for social posts
4. Add one benchmark-like scenario page (burst + stream + undo)

Success criteria:

- Every external post points to a strong “why + proof + install” funnel.

## Phase 3 (Days 26-40): Distribution Sprint

Goals:

- Turn awareness into npm installs
- Establish repeated visibility

Execution:

1. LinkedIn:
   - 4 posts (build story, feature proof, migration, launch)
2. daily.dev:
   - 1 detailed engineering post
3. dev.to/Hashnode:
   - 2 tutorial/comparison posts
4. Product Hunt prep:
   - listing copy, media, first-day comment strategy

Success criteria:

- Steady weekly install growth and inbound GitHub traffic.

## Phase 4 (Days 41-60): Launch + Iteration

Goals:

- Run launch week without breaking trust
- Prioritize iteration from real feedback

Execution:

1. Launch week:
   - Product Hunt + LinkedIn + X + docs banner
2. Community push:
   - selective Show HN / Reddit shares with value-first copy
3. Feedback triage:
   - categorize into `bug`, `DX`, `feature`, `docs`
4. Patch releases:
   - fast fixes for setup friction and regressions

Success criteria:

- Users can explain “why emre-toast” in one sentence after reading the landing page.

## Weekly KPI Dashboard

Track weekly:

- npm installs
- docs unique visitors
- install command copy clicks
- GitHub stars
- issues opened vs resolved
- percentage of users who reach first successful toast

## Distribution Playbook (Where to Post)

Use this order:

1. npm page + GitHub README
2. docs site + demo
3. LinkedIn
4. daily.dev
5. dev.to / Hashnode
6. Product Hunt
7. Show HN / selected Reddit communities

Rule:

- Never post just a link.
- Always include: problem, before/after, concrete example, and install path.
