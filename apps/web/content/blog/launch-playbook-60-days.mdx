If you want real distribution, treat launch like product work. The goal is simple: steady growth for `npm install @emrelutfi/emre-toast`.

## Positioning First

Use one clear positioning line everywhere:

`The React toast library for real-time product teams.`

Avoid vague claims like "best for everyone." Clear positioning converts better.

## Phase 1 (Days 1-10): Trust Baseline

- Ensure README quick start works copy-paste.
- Keep docs and API naming consistent.
- Keep npm page clean with 3-4 practical examples.

## Phase 2 (Days 11-25): Story Assets

- Publish a short product story ("why this exists").
- Publish one competitor comparison post.
- Publish one integration tutorial.
- Prepare 30-45 second product demo video.

## Phase 3 (Days 26-40): Distribution Sprint

- LinkedIn: 4 posts across 2 weeks.
- daily.dev: one technical post with real code and outcomes.
- dev.to/Hashnode: tutorial + migration content.

## Phase 4 (Days 41-60): Launch + Feedback Loop

- Run Product Hunt launch with prepared comments.
- Share one value-first Show HN post.
- Ship quick fixes from launch feedback in patch releases.

## Where to Share

Best order:

1. npm + GitHub README
2. official docs and live demo
3. LinkedIn
4. daily.dev
5. dev.to / Hashnode
6. Product Hunt
7. Show HN and selective Reddit posts

## KPI Checklist

Track weekly:

- npm installs
- docs visits
- install-command copy clicks
- GitHub stars
- issue resolution speed

If installs stay flat, improve messaging and onboarding before shipping new features.
